## 1. Техническая реализация и архитектура (по ТЗ)

В рамках проекта выполнены следующие архитектурные и программные требования:

### Объектно-ориентированная модель (Core)
* **Абстракция и инкапсуляция:** Создан базовый абстрактный класс `Team` (Команда), содержащий основные свойства статистики (`Wins`, `Losses`, `Draws`, `MatchesPlayed`) и абстрактное свойство подсчета очков `CalculatePoints`.
* **Полиморфизм:** Реализованы классы-наследники `FootballTeam`, `BasketballTeam` и `VolleyballTeam` со специфической для каждого вида спорта логикой начисления баллов.
* **Перегрузка операторов:** В базовом классе `Team` перегружены операторы сравнения (`>`, `<`, `>=`, `<=`) для ранжирования команд по очкам, количеству побед и ничьих.

### Обобщенные типы и коллекции (Generics)
* Разработан интерфейс `ITable` и обобщенный класс турнира `TournamentTable<T>`.
* Хранение данных внутри турнира организовано через строго типизированный массив `T[] Teams`.
* Реализован кастомный алгоритм пузырьковой сортировки (Bubble Sort) для упорядочивания элементов массива как по алфавиту, так и по спортивным показателям с применением перегруженных операторов.

### Работа с данными (Data)
* Разработан обобщенный интерфейс `IDataSerializer<T>` и абстрактный класс `BaseSerializer<T>`.
* Реализованы конкретные классы сериализации `JsonTournamentSerializer` и `XmlTournamentSerializer`.
* Настроено автоматическое создание папки и генерация дефолтных наборов данных (JSON-файлов) при первом запуске, если файлы отсутствуют на диске.

## 2. Интерфейс взаимодействия (Руководство пользователя)

Графический интерфейс состоит из двух последовательных окон. Ниже приведена логика работы всех элементов управления, условий их активации и переходов.

### Стартовое окно (Выбор параметров турнира)
При запуске приложения открывается главное меню выбора. Взаимодействие происходит через следующие элементы:
* **Выпадающий список «Выберите вид спорта» (`ComboBox`):** Служит для выбора спортивной дисциплины (Футбол, Баскетбол, Волейбол).
* **Выпадающий список «Выберите номер сезона» (`ComboBox`):** Служит для выбора года проведения турнира (2024, 2025, 2026).
* **Кнопка «Показать таблицу»:**
  * **Логика активации:** По умолчанию кнопка заблокирована (`IsEnabled="False"`). Она автоматически переходит в активное состояние только тогда, когда пользователь выберет значения в **обоих** выпадающих списках.
  * **Переход между окнами:** При нажатии на активную кнопку текущее стартовое окно закрывается (`this.Close()`), параллельно инициализируется, считывает нужный файл с диска и открывается на экране основное окно таблицы (`TournamentWindow`).

### Основное окно (Турнирная таблица)
Открывается автоматически после успешного выбора параметров в главном меню.
* **Таблица данных (`DataGrid`):** Отображает список команд выбранного сезона и их статистику. При первой загрузке данные автоматически сортируются в алфавитном порядке названий команд (базовая сортировка по умолчанию).
* **Кнопка «Отсортировать по имени»:** Всегда активна. При нажатии запускает сортировку пузырьком по названию команды и мгновенно обновляет отображение строк в таблице.
* **Кнопка «Отсортировать по очкам»:** Всегда активна. При нажатии перестраивает строки таблицы сверху вниз (от лидера к аутсайдеру), используя перегруженные операторы сравнения (`>` и `<`) на основе набранных очков, побед и ничьих.
* **Кнопка «Симулировать матч»:** Служит для демонстрации динамики интерфейса. Генерирует случайный результат матча для команд, начисляет им очки, мгновенно обновляет `DataGrid` и автоматически перезаписывает изменения в JSON-файл на диске.
* **Закрытие второй вкладки (окна):** Для закрытия окна таблицы и полного завершения работы приложения используется стандартная системная кнопка закрытия (крестик) в правом верхнем углу окна.

## 3. Структура решения и запуск
* `TournamentTable.Model` — библиотека классов (бизнес-логика, модели команд, слой сериализации).
* `TournamentTable` — исполняемый WPF-проект (визуальные окна XAML и логика UI).

**Запуск:** Собрать решение в IDE и запустить проект `TournamentTable`. Файлы данных будут автоматически созданы по пути размещения сборки: `bin/Debug/TournamentData/`.
* Создан абстрактный класс `Team` (Команда), инкапсулирующий базовые свойства (Название, Победы, Поражения, Ничьи) и абстрактное свойство подсчета очков.
* Реализованы классы-наследники для конкретных видов спорта: `FootballTeam`, `BasketballTeam` и `VolleyballTeam` со своей логикой начисления баллов (полиморфизм).
* Перегружены операторы сравнения (`>`, `<`, `>=`, `<=`) в базовом классе для сравнения команд по очкам, победам и ничьим.

### 2. Обобщенные типы и коллекции (Generics)
* Разработан интерфейс `ITable` и обобщенный класс турнира `TournamentTable<T>`.
* Внутри класса турнира данные хранятся в виде типизированного массива `T[] Teams`.
* Реализована ручная пузырьковая сортировка элементов массива (как по алфавиту, так и по очкам с использованием перегруженных операторов).

### 3. Работа с файловой системой (Data)
* Создан обобщенный интерфейс `IDataSerializer<T>`.
* Реализован абстрактный класс `BaseSerializer<T>`, отвечающий за базовую логику подготовки путей.
* Реализованы классы `JsonTournamentSerializer` и `XmlTournamentSerializer` для сохранения и загрузки данных турнира.
* Настроена автоматическая генерация стартовых данных (JSON-файлов) при первом запуске приложения, если файлы отсутствуют на диске.

### 4. Графический интерфейс (UI)
* **Главное окно:** Содержит выпадающие списки (`ComboBox`) для выбора вида спорта и года сезона. Кнопка перехода к таблице активируется только при выборе обоих параметров.
* **Окно таблицы:** Выводит загруженные из файла данные в компонент `DataGrid`. 
* Реализованы кнопки управления: «Отсортировать по имени» и «Отсортировать по очкам». Обновление таблицы происходит мгновенно после нажатия.

## Структура решения
