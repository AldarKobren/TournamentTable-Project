# Проект: Управление спортивными турнирными таблицами (WPF)

## Описание проекта
Приложение представляет собой автоматизированную систему для ведения, просмотра, сортировки и сравнительного анализа турнирных таблиц по различным видам спорта. Проект содержит интерактивное главное меню и динамические окна отображения результатов. Все данные сохраняются локально в файловую систему.

---

## Выполнение критериев оценивания (Moodle)

### Базовая часть (5 баллов)
* **Абстрактная модель (`Team`):** Базовый класс участников, инкапсулирующий название, количество побед, ничьих, поражений и сыгранных матчей. Содержит абстрактное свойство/метод `CalculatePoints` для вычисления очков.
* **Полиморфизм видов спорта:** Реализованы классы-наследники `FootballTeam`, `BasketballTeam` и `VolleyballTeam`, переопределяющие логику `CalculatePoints` под регламенты своих ассоциаций.
* **Интерфейс `ITable`:** Описывает ключевые контракты системы: метод проведения матча `Match(Team, Team)`, базовую сортировку `SortDefault()` и сортировку по очкам `SortByScore()`.
* **Класс турнира (`TournamentTable`):** Реализует интерфейс `ITable`. Хранит массив команд (`T[] Teams`), массив матчей и метаданные сессии (название, год сезона).
* **Генерация и JSON-сериализация:** Приложение автоматически создает пул из 10–20 уникальных команд. Реализован класс `JsonTournamentSerializer`, считывающий и создающий JSON-файлы при первом запуске программы.
* **Графический интерфейс базовой части:** Главное меню содержит списки выбора спорта и сезона, а также кнопку перехода, которая становится активной только при полной конфигурации параметров.

### Дополнительная часть (3 балла)
* **Архитектурная библиотека `Model`:** Все компоненты бизнес-логики, классы вычислений и алгоритмы сериализации полностью изолированы в проекте-библиотеке `TournamentTable.Model`.
* **Модификатор `partial` для ядра:** Класс `TournamentTable` разделен на части. В дополнительной части реализован метод `Disqual(Team)`, позволяющий исключить команду из дальнейшего участия в кубке.
* **Продвинутый алгоритм ранжирования:** Доработан метод сортировки по очкам `SortByScore()`. При равенстве баллов места распределяются по цепочке дополнительных критериев:
  1. Количество побед (или очки за победы).
  2. Количество ничьих.
  3. Результат личной встречи между командами.
* **Учет разделения мест:** Если у команд совпадают абсолютно все показатели, они делят одинаковое (более высокое) место в UI, а следующее за ними место пропускается (например: 1, 2, 2, 4).

### Продвинутая часть (2 балла)
* **Разделение `Core` и `Data`:** Внутри библиотеки `Model` папки строго разграничены по назначению: в `Core` лежат игровые сущности, а в `Data` — логика сохранения данных.
* **Полиморфная XML/JSON сериализация:** Реализован общий абстрактный класс для сериализации, от которого унаследованы независимые парсеры JSON и XML.
* **Выбор формата на лету:** В главном меню доступен выбор расширения файла. При изменении формата в `ComboBox` приложение мгновенно копирует и конвертирует все данные из текущего файла в файл нового формата.
* **Аналитический блок `Getchanges`:** Добавлена `partial` часть класса `TournamentTable` с методом `Getchanges(TournamentTable)`, который сравнивает текущий сезон с переданным прошлым и возвращает кортеж изменений мест (например: +3, -4, или прочерк для новичков).
* **Окно «Сравнительная таблица»:** В главном меню активируется кнопка, открывающая специальное аналитическое окно, где наглядно выводится динамика изменения позиций команд текущего сезона относительно предыдущего.

## Описание интерфейса взаимодействия (User Guide)

### Главное меню (Выбор сезона)
* **Выпадающие списки параметров (`ComboBox`):** Служат для выбора текущей дисциплины (*Футбол / Баскетбол / Волейбол*) и игрового сезона (*2024 / 2025 / 2026*).
* **Список «Формат сохранения»:** Позволяет переключить режим базы данных между XML и JSON. При смене формата данные автоматически мигрируют в новый файл.
* **Кнопка «Показать таблицу»:** * **Условие активации:** Заблокирована по умолчанию. Становится доступной только после выбора значений во всех обязательных списках конфигурации.
  * **Действие:** При нажатии выводит информационное окно об успешном считывании/создании файла базы данных, закрывает стартовую форму и инициализирует таблицу.
* **Кнопка «Сравнительная таблица»:** Становится активной, когда выбран вид спорта. Открывает отдельное графическое окно, отображающее разницу мест команд между двумя смежными сезонами.

### Окно «Турнирная таблица»
* **Сетка `DataGrid`:** Интерактивное поле, куда выводятся: Место, Название команды, Матчи, Победы, Ничьи, Поражения и Очки. Изначально данные упорядочены в алфавитном порядке.
* **Кнопка «Отсортировать по имени»:** Выстраивает строки по алфавиту с помощью метода `SortDefault()`.
* **Кнопка «Отсортировать по очкам»:** Перестраивает таблицу по убыванию очков и сложным правилам дополнительных критериев.
* **Кнопка «Симулировать матч»:** Автоматически рассчитывает случайный результат встречи между выбранными командами, обновляет сетку UI и перезаписывает изменения на жесткий диск.
* **Закрытие окна таблицы:** Выполняется кликом на стандартный крестик в углу окна, полностью сохраняя игровой прогресс.
